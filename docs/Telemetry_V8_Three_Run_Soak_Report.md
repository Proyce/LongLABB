# LongLAB Telemetry V8 Three-Run Soak Report

## Method

Three simulations ran concurrently for 300 seconds of real wall-clock time, with 50 trades per run and one-second lifecycle observations.

The harness imports the production LongLAB modules for:

- profit-lock calculation and protection synchronization;
- immediate exit evaluation;
- fee-adjusted PnL finalization;
- telemetry compaction;
- V8 CSV/JSON export;
- batch analysis package construction;
- null-safe render guards.

The harness also executes a 15-second fast-scan cadence and a decoupled 60-second deep-telemetry cadence.

## Interpretation

The runs are engineering validation, not alpha evaluation. Their PnL is generated by deterministic market profiles and should not be interpreted as expected strategy performance.

The relevant conclusions are:

1. No null/NaN PnL entered render analytics.
2. No page-level rendering exception was produced.
3. Lifecycle processing continued throughout the 300-second window.
4. Fast scans and deep enrichment remained independent.
5. Mixed SL, lock, timeout, and fallback paths finalized cleanly.
6. Runtime and exported records contained none of the four audited heavy duplicate objects.
